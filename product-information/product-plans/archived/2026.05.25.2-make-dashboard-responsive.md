# Make Dashboard Responsive — Plan

## Overview

The dashboard currently renders correctly on laptop and desktop but breaks on tablet and phone. The approach mirrors the `task-details-panel` pattern: inject `PlatformService`, bind conditional CSS modifier classes (`--mobile`, `--tablet`), and use `@if` for conditional rendering where structural differences are needed.

---

## Problem Areas

| Area | Phone issue | Tablet issue |
|---|---|---|
| Header | Titles + 3-select filter overflow horizontally | Tight but still overflows at narrow widths |
| Stats cards | All 5 in one row → tiny cards | Same |
| Data-viz charts | Two cards side-by-side at 400 px min-height | Same, but more workable |
| Task table | 4 columns too wide; List & Due columns waste space | Manageable with slight padding reduction |
| Container padding | `var(--space-8)` on all sides too large for small screens | `var(--space-4)` would be better |

---

## Changes Per File

### 1. `dashboard.component.ts`
- Inject `PlatformService` and expose it as `readonly platformService`.

### 2. `dashboard.component.html`
- Bind `[class.--mobile]="platformService.isMobile()"` and `[class.--tablet]="platformService.isTablet()"` on `.dashboard-container`.
- The header already contains `app-dashboard-filter` after the titles. The CSS modifier will handle stacking it below the titles on mobile.

### 3. `dashboard.component.scss`

#### `.dashboard-container.--mobile` / `.--tablet`
- Reduce padding to `var(--space-4)` on mobile, `var(--space-6)` on tablet.
- Reduce `gap` to `var(--space-4)`.

#### `.--mobile .header-container` / `.--tablet .header-container`
- `flex-direction: column` so the filter wraps below the title block.
- Remove the fixed `height: 10%` — use `height: auto` instead.

#### `.--mobile .stats-cards-container` / `.--tablet .stats-cards-container`
- Override the single-row `grid-auto-flow: column` with a wrapping 2-column grid:
  - Tablet: `grid-template-columns: repeat(3, 1fr)` (5 cards across 2 rows: 3 + 2).
  - Phone: `grid-template-columns: repeat(2, 1fr)` (5 cards across 3 rows: 2 + 2 + 1).
- Switch `grid-auto-flow` back to `row`.

#### `.--mobile .data-visualizations-row-container` / `.--tablet .data-visualizations-row-container`
- `flex-direction: column` so each chart card stacks vertically.
- Remove the `min-height: 400px` constraint (or reduce to `300px`) so charts don't overflow the viewport.

#### `.--mobile .data-visualization-card`
- Ensure charts have an explicit height (e.g., `300px`) so ApexCharts renders correctly when the parent is no longer flex-sized.

### 4. `dashboard-filter.component.ts`
- Inject `PlatformService` and expose it as `readonly platformService`.

### 5. `dashboard-filter.component.html`
- Bind `[class.--mobile]="platformService.isMobile()"` on `.dashboard-filter-container`.

### 6. `dashboard-filter.component.scss`
- `.dashboard-filter-container.--mobile`:
  - `flex-direction: column`
  - `align-items: stretch`
  - Each `app-select` / `app-multi-select` takes full width.

### 7. `task-table.component.ts`
- Inject `PlatformService` and expose it as `readonly platformService`.

### 8. `task-table.component.html`
- Bind `[class.--mobile]="platformService.isPhone()"` on `.task-table-card`.
- Conditionally render the **List** and **Due** `<th>` / `<td>` columns using `@if (!platformService.isPhone())` so the phone layout shows only Task and Status columns.

### 9. `task-table.component.scss`
- `.task-table-card.--mobile`:
  - Reduce `padding` to `var(--space-4)`.
- Ensure `.col-task` takes up the remaining width when the other columns are hidden.
