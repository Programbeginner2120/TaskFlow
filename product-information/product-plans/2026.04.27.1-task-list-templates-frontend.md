# Task List Templates — Frontend Implementation Plan

## Overview

This plan covers the Angular frontend work required to let users create task list templates. The feature is modelled after the mockups in `product-information/mockup-images/new-template-home-page.png` and `new-template-modal.png`. The backend API is already implemented under `/task-list-templates`.

---

## Mockup Summary

- **Sidebar** — A new "TEMPLATES" section sits below the "LISTS" section. It contains a single `+ New Template` action link. No individual template items are listed in the sidebar at this stage.
- **New Template Modal** — A scrollable modal dialog with:
  - **Template Name** — plain text input, placeholder `e.g. Weekly Sprint Tasks`
  - **Color** — a row of eight circle swatches; the first swatch is selected by default
  - **Tasks** — a dynamic list of tasks, each with a *title* text input and a *Notes (optional)* text input; an `+ Add Task` button appends new blank task rows
  - **Schedule** — three stacked dropdowns: *Schedule type* (`Recurring`), *Frequency* (`Daily` / `Weekly` / `Monthly`), and a context-sensitive third dropdown (`Day of week` when Weekly is selected; `Day of month` when Monthly)
  - **Footer** — `Cancel` (secondary) and `Create Template` (primary) buttons

## Backend API Reference

| Method | Path | Request body | Response |
|--------|------|-------------|----------|
| `GET` | `/task-list-templates` | — | `TaskListTemplateApiResponse[]` |
| `POST` | `/task-list-templates` | `CreateTaskListTemplateRequest` | `TaskListTemplateApiResponse` |
| `PUT` | `/task-list-templates/{id}` | `UpdateTaskListTemplateRequest` | `TaskListTemplateApiResponse` |
| `DELETE` | `/task-list-templates/{id}` | — | `204 No Content` |

### `CreateTaskListTemplateRequest` shape
```json
{
  "name": "Weekly Sprint Tasks",
  "color": "#6366f1",
  "rrule": "FREQ=WEEKLY;BYDAY=MO",

  ]
}
```

### Step 6 — Create the rrule utility

**New file:** `frontend/src/app/utils/rrule.util.ts`

A pure function that builds an RFC 5545 RRULE string from the form values selected in the modal:

```typescript
export type RruleFrequency = 'DAILY' | 'WEEKLY' | 'MONTHLY';
export type RruleDay = 'MO' | 'TU' | 'WE' | 'TH' | 'FR' | 'SA' | 'SU';

export function buildRrule(freq: RruleFrequency, day?: RruleDay, monthDay?: number): string {
    if (freq === 'WEEKLY' && day)   return `FREQ=WEEKLY;BYDAY=${day}`;
    if (freq === 'MONTHLY' && monthDay) return `FREQ=MONTHLY;BYMONTHDAY=${monthDay}`;
    return `FREQ=${freq}`;
}
```

#### Schedule dropdowns

- **Schedule type**: one option — `Recurring`. Display the dropdown as present in the mockup; leave it non-interactive for the MVP since it is the only option.
- **Frequency**: `Daily`, `Weekly`, `Monthly` — drives which third dropdown to show.
- **Third dropdown (conditional)**:
  - When `Weekly`: day-of-week picker (`Monday` … `Sunday`), maps to `MO` … `SU`.
  - When `Monthly`: day-of-month picker (1–28), maps to `BYMONTHDAY`.
  - When `Daily`: hidden (no third dropdown needed).

#### On submit — `createTemplate()`
1. Build the rrule string via `buildRrule()`.
2. Detect timezone with `Intl.DateTimeFormat().resolvedOptions().timeZone`.
3. Filter out task rows with empty titles before sending.
4. Call `TaskListTemplateStateService.addTemplate(...)`.
5. Emit `created` then `close`.
