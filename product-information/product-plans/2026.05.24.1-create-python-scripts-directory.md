## Shared Infrastructure

### `shared/encryption.py` ŌĆö Spring Encryption in Python

The backend uses Spring Security's `Encryptors.text(ENCRYPTION_PASSWORD, ENCRYPTION_SALT)`.
This must be replicated exactly so the Java service can decrypt the values the scripts insert.

**Spring Security `Encryptors.text()` internals:**
- Key derivation: **PBKDF2-HMAC-SHA1**, 1024 iterations, 256-bit (32-byte) key.
- PBKDF2 salt: `bytes.fromhex(ENCRYPTION_SALT)` ŌĆö the fixed per-deployment salt from env.
- Cipher: **AES-256 CBC, PKCS5 padding** (PKCS5 == PKCS7 for AES block size).
- Per-message random **IV: 8 bytes** (`KeyGenerators.secureRandom(8)` in the Java code).
- Output format: `hex(iv) + hex(ciphertext)` ŌĆö a single lowercase hex string, no separator.

> **Verification note:** Spring's `AesBytesEncryptor` in CBC mode typically requires a
> 16-byte IV, but the library is configured with an 8-byte generator. During implementation,
> verify by encrypting a known string in Python and confirming the Java backend decrypts it
> successfully (or vice-versa). If decryption fails, adjust `_IV_BYTES` from 8 to 16.

```python
# shared/encryption.py

import hashlib
import secrets

from cryptography.hazmat.backends import default_backend
from cryptography.hazmat.primitives import padding as sym_padding
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes

from shared.env_loader import EncryptionConfig

_IV_BYTES  = 8   # matches Spring's KeyGenerators.secureRandom(8) ŌĆö verify during impl
_KEY_BYTES = 32  # AES-256


class SpringTextEncryptor:
    """
    Python equivalent of Spring Security's Encryptors.text(password, salt).

    Thread-safe: key derivation happens once at construction time; each
    encrypt/decrypt call derives its own Cipher instance.
    """

    def __init__(self, config: EncryptionConfig):
        pbkdf2_salt = bytes.fromhex(config.salt)
        self._key = hashlib.pbkdf2_hmac(
            hash_name="sha1",
            password=config.password.encode("utf-8"),
            salt=pbkdf2_salt,
            iterations=1024,
            dklen=_KEY_BYTES,
        )

    def encrypt(self, plaintext: str) -> str:
        iv      = secrets.token_bytes(_IV_BYTES)
        padder  = sym_padding.PKCS7(128).padder()
        padded  = padder.update(plaintext.encode("utf-8")) + padder.finalize()
        cipher  = Cipher(algorithms.AES(self._key), modes.CBC(iv), backend=default_backend())
        enc     = cipher.encryptor()
        ciphertext = enc.update(padded) + enc.finalize()
        return (iv + ciphertext).hex()

    def decrypt(self, hex_value: str) -> str:
        data       = bytes.fromhex(hex_value)
        iv         = data[:_IV_BYTES]
        ciphertext = data[_IV_BYTES:]
        cipher     = Cipher(algorithms.AES(self._key), modes.CBC(iv), backend=default_backend())
        dec        = cipher.decryptor()
        padded     = dec.update(ciphertext) + dec.finalize()
        unpadder   = sym_padding.PKCS7(128).unpadder()
        return (unpadder.update(padded) + unpadder.finalize()).decode("utf-8")
```

### `utils/random_data.py` ŌĆö Shared Random Data Helpers

Small helpers so each script doesn't import `faker` inline and so constants are shared.

```python
# utils/random_data.py

import secrets
import string

from faker import Faker

_fake = Faker()

LIST_COLORS = [
    "#6366f1", "#ec4899", "#f59e0b",
    "#10b981", "#3b82f6", "#8b5cf6", "#ef4444",
]


def random_color() -> str:
    return secrets.choice(LIST_COLORS)


def random_email() -> str:
    """Generates a disposable-style email that won't clash with real accounts."""
    return f"{_fake.user_name()}{secrets.randbelow(10_000)}@example.com"


def random_password(length: int = 14) -> str:
    alphabet = string.ascii_letters + string.digits + "!@#$%"
    return "".join(secrets.choice(alphabet) for _ in range(length))


def random_list_name() -> str:
    return _fake.bs().title()  # e.g. "Streamline Web-Readiness"


def random_task_title() -> str:
    return _fake.sentence(nb_words=5).rstrip(".")


def random_task_notes() -> str | None:
    """Returns a short paragraph ~60 % of the time, None otherwise."""
    if secrets.randbelow(10) < 6:
        return _fake.paragraph(nb_sentences=2)
    return None
```

## Script 1: `user/create_user.py`

**Purpose:** Insert one user with a random email address, random password, and
`email_verified = TRUE`. Print the credentials so they can be copied and used in
the browser / API client.

**Key points:**
- Password is hashed with `bcrypt` before inserting into `password_hash`.
- `display_name` is set to the username part of the email.
- Returns a `dict` with `id`, `email`, and the plaintext `password` so callers
  can chain this into the other seed scripts.

```python
# user/create_user.py

"""
Usage:
    python -m user.create_user
"""

import bcrypt

from shared.db import get_cursor
from utils.random_data import random_email, random_password


def create_user() -> dict:
    email        = random_email()
    display_name = email.split("@")[0]
    password     = random_password()
    password_hash = bcrypt.hashpw(
        password.encode("utf-8"), bcrypt.gensalt()
    ).decode("utf-8")

    with get_cursor() as cur:
        cur.execute(
            """
            INSERT INTO users (email, password_hash, display_name, email_verified)
            VALUES (%s, %s, %s, TRUE)
            RETURNING id
            """,
            (email, password_hash, display_name),
        )
        user_id = cur.fetchone()[0]

    print("=" * 50)
    print(f"  Created user")
    print(f"  id:       {user_id}")
    print(f"  email:    {email}")
    print(f"  password: {password}")
    print("=" * 50)

    return {"id": user_id, "email": email, "password": password}


if __name__ == "__main__":
    create_user()
```

## Script 2: `task_lists/create_task_lists.py`

**Purpose:** Insert `count` random task lists for a given `user_id`. List names are
encrypted using `SpringTextEncryptor` before being written to the database.

**Key points:**
- `name` column is AES-256 encrypted ŌĆö plaintext is logged, ciphertext is stored.
- Accepts `--user-id` and optional `--count` (default 5) via CLI args.
- Returns the list of newly created list IDs so callers can chain into task creation.

```python
# task_lists/create_task_lists.py

"""
Usage:
    python -m task_lists.create_task_lists --user-id 42
    python -m task_lists.create_task_lists --user-id 42 --count 8
"""

import argparse

from shared.db import get_cursor
from shared.encryption import SpringTextEncryptor
from shared.env_loader import EncryptionConfig
from utils.random_data import random_color, random_list_name


def create_task_lists(user_id: int, count: int = 5) -> list[int]:
    encryptor = SpringTextEncryptor(EncryptionConfig())
    list_ids: list[int] = []

    with get_cursor() as cur:
        for _ in range(count):
            name_plain = random_list_name()
            name_enc   = encryptor.encrypt(name_plain)
            color      = random_color()

            cur.execute(
                """
                INSERT INTO task_lists (user_id, name, color)
                VALUES (%s, %s, %s)
                RETURNING id
                """,
                (user_id, name_enc, color),
            )
            list_id = cur.fetchone()[0]
            list_ids.append(list_id)
            print(f"  Created task list ŌĆö id: {list_id}, name: '{name_plain}', color: {color}")

    return list_ids


if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Create random task lists for a user.")
    parser.add_argument("--user-id", type=int, required=True, help="Target user ID")
    parser.add_argument("--count",   type=int, default=5,    help="Number of lists to create")
    args = parser.parse_args()
    create_task_lists(args.user_id, args.count)
```

## Script 3: `tasks/create_tasks.py`

**Purpose:** Insert `count` random tasks for a given `user_id`. Each task is randomly
assigned to one of the user's lists, or to no list at all.

**Key points:**
- `title` and `notes` are AES-256 encrypted (same encryptor as task lists).
- When assigning to a list, `position` = current task count for that list. This is
  computed with `SELECT COUNT(*) FROM tasks WHERE list_id = %s` **inside the same
  open cursor/transaction** as the subsequent `INSERT`. PostgreSQL reads-your-own-writes
  within a transaction, so sequential inserts within a single `with get_cursor()` block
  will produce monotonically increasing, correct positions.
- When `list_id` is `NULL`, `position` is also `NULL`.
- The candidate pool weights `None` (no list) at roughly 30% by including one `None`
  entry per list. If the user has no lists, every task is created without a list.

```python
# tasks/create_tasks.py

"""
Usage:
    python -m tasks.create_tasks --user-id 42
    python -m tasks.create_tasks --user-id 42 --count 30
"""

import argparse
import random

from shared.db import get_cursor
from shared.encryption import SpringTextEncryptor
from shared.env_loader import EncryptionConfig
from utils.random_data import random_task_notes, random_task_title


def _fetch_list_ids(cur, user_id: int) -> list[int]:
    cur.execute("SELECT id FROM task_lists WHERE user_id = %s", (user_id,))
    return [row[0] for row in cur.fetchall()]


def _next_position(cur, list_id: int) -> int:
    """Returns the current task count for list_id ŌĆö used as the new task's position."""
    cur.execute("SELECT COUNT(*) FROM tasks WHERE list_id = %s", (list_id,))
    return cur.fetchone()[0]


def create_tasks(user_id: int, count: int = 20) -> None:
    encryptor = SpringTextEncryptor(EncryptionConfig())

    with get_cursor() as cur:
        list_ids = _fetch_list_ids(cur, user_id)

        # Candidate pool: each real list_id + one None entry per list (Ōēł30% no-list)
        # If the user has no lists, every task gets list_id = NULL.
        candidates: list[int | None] = list_ids + [None] * max(1, len(list_ids))

        for _ in range(count):
            list_id  = random.choice(candidates)
            position = _next_position(cur, list_id) if list_id is not None else None

            title_enc = encryptor.encrypt(random_task_title())
            notes_raw = random_task_notes()
            notes_enc = encryptor.encrypt(notes_raw) if notes_raw is not None else None

            cur.execute(
                """
                INSERT INTO tasks (user_id, list_id, title, notes, position)
                VALUES (%s, %s, %s, %s, %s)
                RETURNING id
                """,
                (user_id, list_id, title_enc, notes_enc, position),
            )
            task_id = cur.fetchone()[0]
            print(f"  Created task ŌĆö id: {task_id}, list_id: {list_id}, position: {position}")


if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Create random tasks for a user.")
    parser.add_argument("--user-id", type=int, required=True, help="Target user ID")
    parser.add_argument("--count",   type=int, default=20,   help="Number of tasks to create")
    args = parser.parse_args()
    create_tasks(args.user_id, args.count)
```

## Encryption Verification Step

After implementing `shared/encryption.py`, confirm the Python ciphertext is compatible
with the Java backend before running scripts at scale:

1. Encrypt a known string in Python:
   ```python
   from shared.encryption import SpringTextEncryptor
   from shared.env_loader import EncryptionConfig
   enc = SpringTextEncryptor(EncryptionConfig())
   print(enc.encrypt("hello world"))
   ```
2. Insert it into `task_lists.name` manually.
3. Fetch that task list through the running backend API and confirm the name returns as
   `"hello world"`.
4. If decryption fails on the Java side, adjust `_IV_BYTES` in `encryption.py`
   from `8` to `16` and re-test (the discrepancy stems from whether the Spring
   `AesBytesEncryptor` uses an 8-byte or 16-byte IV in the installed version).
