### Step 11 — Create changelog: `20260412.1-CreateTaskListTemplatesTable.sql`

```sql
--liquibase formatted sql
--changeset mattk:20260412.1-CreateTaskListTemplatesTable endDelimiter:go

CREATE TABLE task_list_templates (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT       NOT NULL,
    name            TEXT         NOT NULL,
    color           VARCHAR(7)   NOT NULL DEFAULT '#6366f1',
    rrule           TEXT         NOT NULL,
    timezone        TEXT         NOT NULL DEFAULT 'UTC',
    last_generated  TIMESTAMPTZ,
    next_generate   TIMESTAMPTZ  NOT NULL,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- name is AES-256-GCM encrypted and base64-encoded at the application layer.

ALTER TABLE task_list_templates
    ADD CONSTRAINT fk_task_list_templates_user_id
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE;

-- Scheduler polls this index to find templates that are due
CREATE INDEX idx_task_list_templates_next_generate ON task_list_templates(next_generate);
CREATE INDEX idx_task_list_templates_user_id ON task_list_templates(user_id);
```

Column notes:
- `rrule` — RFC 5545 recurrence rule string (e.g. `FREQ=WEEKLY;BYDAY=MO`). Evaluated by ical4j.
- `timezone` — IANA timezone ID (e.g. `America/New_York`). Stored separately from the RRULE for easy display/editing in the UI.
- `last_generated` — nullable; null means the template has never been generated.
- `next_generate` — set on creation and updated after every generation. The scheduler queries `WHERE next_generate <= NOW()`.

### Step 21 — Create service

`components/tasklisttemplate/service/TaskListTemplateService.java`

Key responsibilities:
- **CRUD** — create, update, delete templates with their nested task and subtask templates. On creation/update of `rrule` or `timezone`, always call `computeNextGenerate()` and persist the result.
- **`computeNextGenerate(String rrule, String timezone) → OffsetDateTime`** — uses ical4j to parse the RRULE string, combined with the given timezone, and returns the next occurrence after `now`. This is the single place where ical4j is called.
- **`generateFromTemplate(TaskListTemplate template) → TaskList`** — creates a real `task_list`, `tasks`, and `subtasks` from the template. Resolves `due_date_offset` to an actual `due_date` (generation date + offset days). Updates `last_generated = NOW()` and `next_generate = computeNextGenerate(...)` on the template after generation.

### Step 23 — Create the internal trigger endpoint

**Background:** `@Scheduled` is not viable on a hosting free tier because the web service spins down after inactivity, meaning the JVM is not running when the trigger would fire. The solution is a secured HTTP endpoint that an external scheduler calls every minute to trigger generation.

**Triggering options (choose one, configured externally — no backend code changes needed per option):**
- **[cron-job.org](https://cron-job.org)** — free, supports 1-minute intervals, sends an HTTP request to your URL on schedule. Recommended for free tier.
- **Render Cron Job** — paid ($1/month minimum), runs a Linux command (`curl ...`) on a schedule. Simple if already on a paid Render plan.
- **GitHub Actions** — free, scheduled workflow using `on: schedule` with a `curl` step. Slightly more setup but no additional service dependency.

**Backend — add `CRON_SECRET` to environment config:**

Add to `env.example` and all `docker-compose*.yml` / Render environment variables:
```
CRON_SECRET=<random-secret-value>
```

Add to `application.yaml`:
```yaml
app:
  cron:
    secret: ${CRON_SECRET}
```

**Backend — create the endpoint:**

`components/tasklisttemplate/controller/TemplateGenerationController.java`

```java
@RestController
@RequiredArgsConstructor
public class TemplateGenerationController {

    @Value("${app.cron.secret}")
    private String cronSecret;

    private final TaskListTemplateService taskListTemplateService;

    @PostMapping("/api/internal/generate-templates")
    public ResponseEntity<Void> generateTemplates(
            @RequestHeader("X-Cron-Secret") String providedSecret) {

        if (!cronSecret.equals(providedSecret)) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }

        taskListTemplateService.generateDueTemplates();
        return ResponseEntity.ok().build();
    }
}
```

Extract the generation loop from the service into a `generateDueTemplates()` method on `TaskListTemplateService` (called here and also callable in tests):

```java
public void generateDueTemplates() {
    List<TaskListTemplate> due = taskListTemplateRepository
        .findDueTemplates(OffsetDateTime.now(ZoneOffset.UTC));
    for (TaskListTemplate template : due) {
        generateFromTemplate(template);
    }
}
```

**Security note:** The endpoint must be excluded from Spring Security authentication (permit the path in the security config) since it uses its own secret-based auth. The `X-Cron-Secret` header value must match the `CRON_SECRET` env var exactly — use a long random value (32+ characters).
