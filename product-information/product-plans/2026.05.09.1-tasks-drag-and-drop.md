## Step 6 — Move Task (Reorder)

Add a new `moveTask(Long userId, Long taskId, Long newPosition)` method to `TaskService`. The existing `updateTask` is not appropriate for this since it carries the full task body and has no concept of positional intent.

**Algorithm:**

- Fetch the task and validate ownership
- If the task has no `listId` or `position`, throw — it's not in an ordered list
- Fetch all sibling tasks via `taskRepository.findByTaskListId(listId)`
- Clamp `newPosition` to `[0, siblings.size() - 1]`
- If moving **down** (`oldPos < newPos`): shift siblings in the range `(oldPos, newPos]` by **-1**
- If moving **up** (`oldPos > newPos`): shift siblings in the range `[newPos, oldPos)` by **+1**
- Set the moved task's position to `newPosition` and persist

**No new endpoint needed.** Add an optional `Long newPosition` field to `UpdateTaskRequest`. In `updateTask`, if `newPosition` is present and differs from the current position, run the sibling-shift logic before persisting.

**Files to change:**
- `UpdateTaskRequest.java` — add optional `Long newPosition` (no `@NotNull`)
- `TaskService.java` — extend `updateTask` with sibling-shift logic when `newPosition` is set
- `TaskRepository.java` — no changes needed (`findByTaskListId` already exists)

## Step 7 — Delete Task (Close the Gap)

Extend the existing `deleteTask(Long userId, Long taskId)` method in `TaskService`.

**Algorithm:**

- Fetch the task and validate ownership (already done)
- If the task has a `listId` and a non-null `position`:
  - Fetch all siblings via `taskRepository.findByTaskListId(listId)`
