# Add Task List Generation Title with Date Format Field

## Overview

Add an optional `generationTitle` field to task list templates. This field defines the name given to a task list when it is generated from the template. It supports date format tokens (e.g. `YYYY-MM-DD`) that are resolved to the actual generation date at generation time. If left blank, generation falls back to the template `name`.

---

## Supported Date Tokens

The following tokens are resolved using the template's configured timezone at the moment of generation:

| Token  | Output               |
|--------|----------------------|
| `YYYY` | 4-digit year         |
| `MM`   | 2-digit month (01–12)|
| `DD`   | 2-digit day (01–31)  |

**Example:** `Weekly Tasks YYYY-MM-DD` → `Weekly Tasks 2026-05-10`

---

## Database

### New migration: `20260510.1-AddGenerationTitleToTaskListTemplates.sql`

Add a nullable `generation_title` column to `task_list_templates`. Nullable because existing records will not have a value and should fall back to `name` at generation time. Like `name`, this column stores AES-256-GCM encrypted, base64-encoded content at the application layer.

```sql
ALTER TABLE task_list_templates
    ADD COLUMN generation_title TEXT;

-- generation_title is AES-256-GCM encrypted and base64-encoded at the application layer.
```

### 5. `TaskListTemplateEncryptionHelper.java`
Extend encrypt/decrypt logic to include `generationTitle`, handling `null` safely (no encryption if null).

### 6. MyBatis generated model (`TaskListTemplateDb`)
The `generation_title` column will need to be reflected in the MyBatis generator config and regenerated, or the field added manually. Regenerate with `./gradlew mbGenerator` (or equivalent) after the migration runs.

### 7. `TemplateGeneratorService.java`
When building the `TaskList` to save, resolve the generation title:
- If `template.getGenerationTitle()` is non-null and non-blank, use it after substituting date tokens with the generation date in the template's timezone.
- Otherwise, fall back to `template.getName()`.

Extract token resolution into a private helper method `resolveGenerationTitle(String pattern, LocalDate date)`:
```java
private String resolveGenerationTitle(String pattern, LocalDate date) {
    return pattern
        .replace("YYYY", String.format("%04d", date.getYear()))
        .replace("MM",   String.format("%02d", date.getMonthValue()))
        .replace("DD",   String.format("%02d", date.getDayOfMonth()));
}
```

Use it in `generateFromTemplate`:
```java
String listName = (template.getGenerationTitle() != null && !template.getGenerationTitle().isBlank())
    ? resolveGenerationTitle(template.getGenerationTitle(), today)
    : template.getName();

TaskList taskList = TaskList.builder()
    .userId(template.getUserId())
    .name(listName)
    ...
```

## Frontend

### 1. `task.interface.ts`
Add `generationTitle` to the relevant interfaces:

- `TaskListTemplate` → `generationTitle: string | null`
- `TaskListTemplateApiResponse` → `generationTitle: string | null`
- `CreateTaskListTemplateRequest` → `generationTitle: string | null`
- `UpdateTaskListTemplateRequest` → `generationTitle: string | null`

### 2. `new-template-modal.component.ts`
- Add signal: `readonly templateGenerationTitle = signal<string>('');`
- Add a `computed` preview signal that resolves date tokens against today's date for display in the UI:
  ```ts
  readonly generationTitlePreview = computed<string>(() => {
      const title = this.templateGenerationTitle().trim();
      if (!title) return '';
      const today = new Date();
      return title
          .replace('YYYY', String(today.getFullYear()).padStart(4, '0'))
          .replace('MM',   String(today.getMonth() + 1).padStart(2, '0'))
          .replace('DD',   String(today.getDate()).padStart(2, '0'));
  });
  ```
- Include `generationTitle` in the `CreateTaskListTemplateRequest` and `UpdateTaskListTemplateRequest` payloads built in `onSubmit()` (send `null` if empty).
- Populate `templateGenerationTitle` from `template().generationTitle` in `ngOnInit` when in edit mode.

### 3. `new-template-modal.component.html`
Add an input field below the template name field:

```html
<app-input
    class="new-template-generation-title"
    [label]="'Generation Title (optional)'"
    [type]="'text'"
    [placeholder]="'e.g. Weekly Tasks YYYY-MM-DD'"
    [(value)]="templateGenerationTitle"
></app-input>
@if (generationTitlePreview()) {
    <p class="generation-title-preview">Preview: {{ generationTitlePreview() }}</p>
}
```

Add a small helper/hint text below the field explaining the supported tokens (`YYYY`, `MM`, `DD`). The preview paragraph only renders when there is content to show.

### 4. `new-template-modal.component.scss`
Add minimal styles for `.generation-title-preview` (muted text, small font size).

## Data Flow Summary

```
User fills in generationTitle ("Tasks YYYY-MM-DD")
        ↓
Saved encrypted in task_list_templates.generation_title
        ↓
Scheduler triggers generateFromTemplate()
        ↓
TemplateGeneratorService decrypts generationTitle, resolves tokens using generation LocalDate + timezone
        ↓
TaskList saved with resolved name ("Tasks 2026-05-10")
```
