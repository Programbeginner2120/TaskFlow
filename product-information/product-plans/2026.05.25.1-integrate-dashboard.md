# Dashboard Integration Plan

## 1. Overview

The dashboard currently displays hardcoded placeholder data across all its charts and the task table. The backend `POST /dashboard-analytics` endpoint already exists and accepts filter parameters. This plan wires everything together: defining frontend request/response types, creating an HTTP service, and driving all visualizations from real task data.

---

## 2. Dashboard State Service — Analysis & Recommendation

**Recommendation: No dedicated state service.**

Rationale for a state service generally: `TaskStateService` and `TaskListStateService` both exist because they cache data that is consumed by many independent components across the app (task list view, task details panel, filter dropdowns, creation modals, etc.).

The dashboard analytics use case differs on every relevant axis:

| Factor | TaskStateService / TaskListStateService | Dashboard Analytics |
|---|---|---|
| Consumers | Multiple unrelated components | Only `DashboardComponent` |
| Data lifetime | Long-lived cache, mutated incrementally | Valid only for current filter selection |
| Shared across routes | Yes | No |
| Parameterized by caller | No — always all user data | Yes — by duration, status, list IDs |

Adding a state service here would require it to hold the filter parameters *and* the response, creating an awkward coupling. Since the analytics data is a pure function of the current filters, it belongs inline in `DashboardComponent` as a reactive fetch triggered by filter changes.

## 3. Backend Changes Required

### 3.1 Fix `DashboardAnalyticsResponse` to use `TaskResponse`

Currently `DashboardAnalyticsResponse` holds `List<Task>` — the raw domain model. This is wrong: domain models should not leak out as API payloads. Change it to `List<TaskResponse>`, which is the established DTO contract used by every other task endpoint.

**`DashboardAnalyticsResponse.java`** — change field:
```java
// Before
private List<Task> tasks;

// After
private List<TaskResponse> tasks;
```
Update the import accordingly (`Task` → `TaskResponse`).

### 3.2 Fix encryption bug in `TaskService.retrieveDashboardAnalyticsData()`

The current method calls `taskRepository.retrieveDashboardAnalyticsTaskData()` and puts the result directly into the response **without decrypting**. Task titles and notes are stored encrypted; this means the endpoint is currently returning ciphertext to the client.

Fix by decrypting and mapping to `TaskResponse` in `DashboardAnalyticsService` (see §3.3), and changing `TaskService` to return a decrypted `List<Task>`:

**`TaskService.java`** — change return type and add decryption:
```java
// Before
public DashboardAnalyticsResponse retrieveDashboardAnalyticsData(Long userId, DashboardAnalyticsRequest request) {
    return DashboardAnalyticsResponse.builder()
            .tasks(this.taskRepository.retrieveDashboardAnalyticsTaskData(userId, request).get())
            .build();
}

// After
public List<Task> retrieveDashboardAnalyticsData(Long userId, DashboardAnalyticsRequest request) {
    return this.taskRepository.retrieveDashboardAnalyticsTaskData(userId, request).get()
            .stream()
            .map(encryptionHelper::decryptTask)
            .toList();
}
```

### 3.3 Map `Task` → `TaskResponse` in `DashboardAnalyticsService`

`DashboardAnalyticsService` injects `TaskConverter` (already a Spring bean via MapStruct) and performs the mapping, mirroring how `TaskController` handles it. `TaskConverter.toResponse()` already derives `completed = (completedAt != null)` via its MapStruct expression.

**`DashboardAnalyticsService.java`**:
```java
// Add to fields
private final TaskConverter taskConverter;

// Updated method
public DashboardAnalyticsResponse retrieveDashboardAnalyticsData(Long userId, DashboardAnalyticsRequest request) {
    List<TaskResponse> tasks = taskService.retrieveDashboardAnalyticsData(userId, request)
            .stream()
            .map(taskConverter::toResponse)
            .toList();
    log.debug("Returning {} tasks for user {}", tasks.size(), userId);
    return DashboardAnalyticsResponse.builder()
            .tasks(tasks)
            .build();
}
```

## 4. Step 1 — Update `dashboard.interface.ts`

With the backend now returning `List<TaskResponse>`, the serialized API shape matches the existing `TaskApiResponse` interface exactly — **no new API response type is needed**. Only the request types and the analytics response wrapper need to be added.

**File:** `src/app/interfaces/dashboard.interface.ts`

```typescript
import { LucideIconData } from "lucide-angular";
import { TaskApiResponse } from './task.interface';

// ─── Existing interfaces (unchanged) ────────────────────────────────────────

export interface StatisticsCard {
    label: string;
    value: number;
    iconBackgroundColor: string;
    icon: LucideIconData;
}

export type TaskStatus = 'Active' | 'Completed' | 'Overdue';

export interface TaskTableRow {
    title: string;
    listName: string | null;
    listColor: string | null;
    dueDate: Date | null;
    status: TaskStatus;
}

// ─── Analytics request / response ───────────────────────────────────────────

export type TaskDataDuration = 'LAST_7_DAYS' | 'LAST_30_DAYS' | 'LAST_90_DAYS' | 'ALL_TIME';
export type TaskDataStatus   = 'ACTIVE' | 'COMPLETED' | 'ALL';

export interface DashboardAnalyticsRequest {
    durationSelection: TaskDataDuration;
    statusSelection:   TaskDataStatus;
    listSelections:    number[];
}

// TaskResponse serializes to the same shape as TaskApiResponse — reuse it directly.
export interface DashboardAnalyticsApiResponse {
    tasks: TaskApiResponse[];
}
```

No separate `DashboardTask` interface is needed — `DashboardComponent` uses the existing `Task` domain type after mapping via the existing `toTask()` mapper.

## 5. Step 2 — Create `dashboard-analytics.service.ts`

Plain HTTP service — no caching, no state.

**File:** `src/app/services/dashboard-analytics.service.ts`

```typescript
import { HttpClient } from '@angular/common/http';
import { inject, Injectable } from '@angular/core';
import { Observable } from 'rxjs';
import {
    DashboardAnalyticsRequest,
    DashboardAnalyticsApiResponse,
} from '../interfaces/dashboard.interface';

@Injectable({ providedIn: 'root' })
export class DashboardAnalyticsService {

    private readonly http = inject(HttpClient);
    private readonly API = '/dashboard-analytics';

    getAnalytics(request: DashboardAnalyticsRequest): Observable<DashboardAnalyticsApiResponse> {
        return this.http.post<DashboardAnalyticsApiResponse>(this.API, request);
    }
}
```

## 7. Step 4 — Update `DashboardComponent`

This is the main integration step. Replace all hardcoded placeholder signals with:
1. A reactive fetch triggered by filter changes.
2. `computed()` signals that derive every visual from the fetched tasks.

### 7.1 Fetch strategy

Use `toObservable()` + `switchMap` from `@angular/core/rxjs-interop` to reactively fetch when any filter changes. The derived tasks are stored in a writable signal updated by the subscription.

### 7.2 Derived data computations

All chart signals become `computed()` signals derived from the raw tasks signal and the task lists from `TaskListStateService`.

| Visual | Derived from |
|---|---|
| Statistics cards (5) | Count total, completed (`completedAt != null`), active, overdue (`dueDate < today && !completed`), due today (`dueDate === today && !completed`) |
| Progress gauge | `Math.round(completed / total * 100)` |
| Donut chart | Group tasks by `listId` → count per list (name from `TaskListStateService`) |
| Bar chart | Per list: active count vs completed count (two series, categories = list names) |
| Multi-line chart | Bucket `createdAt` and `completedAt` into date intervals based on duration filter (two series: "Created", "Completed") |
| Task table rows | Map each `Task` → `TaskTableRow` (resolve list name/color from `TaskListStateService`) |

### 7.3 Date bucketing for the line chart

| Duration | Bucket size | Max buckets |
|---|---|---|
| LAST_7_DAYS | 1 day | 7 |
| LAST_30_DAYS | 1 day | 30 |
| LAST_90_DAYS | 1 week | ~13 |
| ALL_TIME | 1 month | dynamic |

### 7.4 Component skeleton

```typescript
import {
    ChangeDetectionStrategy, Component, DestroyRef,
    computed, inject, signal, effect
} from '@angular/core';
import { takeUntilDestroyed, toObservable } from '@angular/core/rxjs-interop';
import { combineLatest, switchMap, catchError, of } from 'rxjs';
import { DashboardAnalyticsService } from '../../services/dashboard-analytics.service';
import { TaskListStateService } from '../../services/task-list-state.service';
import { toDashboardTask } from '../../mappers/dashboard-analytics.mapper';
import { Task } from '../../interfaces/task.interface';
import {
    TaskDataDuration, TaskDataStatus,
    StatisticsCard, TaskTableRow
} from '../../interfaces/dashboard.interface';
import { DonutSlice, BarSeriesData, LineSeriesData } from '../../interfaces/charts.interface';
// ... chart/component imports unchanged ...

@Component({
    selector: 'app-dashboard',
    templateUrl: './dashboard.component.html',
    styleUrls: ['./dashboard.component.scss'],
    imports: [ /* unchanged chart + filter + table imports */ ],
    changeDetection: ChangeDetectionStrategy.OnPush,
})
export class DashboardComponent {

    private readonly analyticsService   = inject(DashboardAnalyticsService);
    private readonly taskListStateService = inject(TaskListStateService);
    private readonly destroyRef         = inject(DestroyRef);

    // Filter state (unchanged — already bound to DashboardFilterComponent)
    readonly filterDuration  = signal<TaskDataDuration>('LAST_7_DAYS');
    readonly filterStatus    = signal<TaskDataStatus>('ALL');
    readonly filterListIds   = signal<number[]>([]);

    // Raw fetched tasks
    private readonly _tasks   = signal<Task[]>([]);
    readonly loading          = signal(false);

    // ─── Reactive fetch ───────────────────────────────────────────────────────

    constructor() {
        combineLatest([
            toObservable(this.filterDuration),
            toObservable(this.filterStatus),
            toObservable(this.filterListIds),
        ]).pipe(
            switchMap(([duration, status, listIds]) => {
                this.loading.set(true);
                return this.analyticsService.getAnalytics({
                    durationSelection: duration,
                    statusSelection:   status,
                    listSelections:    listIds,
                }).pipe(catchError(() => of({ tasks: [] })));
            }),
            takeUntilDestroyed(this.destroyRef),
        ).subscribe(response => {
            this._tasks.set(response.tasks.map(toDashboardTask));
            this.loading.set(false);
        });
    }

    // ─── Computed signals ─────────────────────────────────────────────────────

    private readonly today = new Date();
    private readonly todayStr = this.today.toDateString();

    readonly statisticsCards = computed<StatisticsCard[]>(() => {
        const tasks = this._tasks();
        const total     = tasks.length;
        const completed = tasks.filter(t => t.completed).length;
        const active    = tasks.filter(t => !t.completed).length;
        const overdue   = tasks.filter(t =>
            !t.completed && t.dueDate !== null && t.dueDate < this.today
        ).length;
        const dueToday  = tasks.filter(t =>
            !t.completed && t.dueDate !== null && t.dueDate.toDateString() === this.todayStr
        ).length;
        return [
            { label: 'Total',     value: total,     iconBackgroundColor: '#EFEFFD', icon: ListTodo    },
            { label: 'Completed', value: completed,  iconBackgroundColor: '#E8F9EE', icon: CheckCircle2 },
            { label: 'Active',    value: active,     iconBackgroundColor: '#EFEFFD', icon: Activity    },
            { label: 'Overdue',   value: overdue,    iconBackgroundColor: '#FDECEC', icon: AlertTriangle },
            { label: 'Due Today', value: dueToday,   iconBackgroundColor: '#FDF5E6', icon: Clock       },
        ];
    });

    readonly completionRate = computed(() => {
        const tasks = this._tasks();
        if (tasks.length === 0) return 0;
        return Math.round(tasks.filter(t => t.completed).length / tasks.length * 100);
    });

    readonly tasksByList = computed<DonutSlice[]>(() => {
        const tasks = this._tasks();
        const lists = this.taskListStateService.lists();
        const counts = new Map<number | null, number>();
        for (const t of tasks) {
            counts.set(t.listId, (counts.get(t.listId) ?? 0) + 1);
        }
        return Array.from(counts.entries()).map(([listId, value]) => ({
            name:  lists.find(l => l.id === listId)?.name ?? 'No List',
            value,
        }));
    });

    readonly activeVsCompletedByList = computed<{ categories: string[]; series: BarSeriesData[] }>(() => {
        const tasks = this._tasks();
        const lists = this.taskListStateService.lists();
        const categories = lists.map(l => l.name);
        const activeCounts    = lists.map(l => tasks.filter(t => t.listId === l.id && !t.completed).length);
        const completedCounts = lists.map(l => tasks.filter(t => t.listId === l.id &&  t.completed).length);
        return {
            categories,
            series: [
                { name: 'Active',    data: activeCounts    },
                { name: 'Completed', data: completedCounts },
            ],
        };
    });

    readonly createdVsCompletedOverTime = computed<{ categories: string[]; series: LineSeriesData[] }>(() => {
        // Build date buckets based on the current duration filter, then count
        // tasks whose createdAt / completedAt falls in each bucket.
        // Returns { categories: string[], series: LineSeriesData[] }
        // (implementation detail: see Step 4 notes on bucket sizing)
        return buildTimeSeries(this._tasks(), this.filterDuration());
    });

    readonly taskTableRows = computed<TaskTableRow[]>(() => {
        const tasks = this._tasks();
        const lists = this.taskListStateService.lists();
        return tasks.map(t => {
            const list = lists.find(l => l.id === t.listId);
            const status: TaskStatus =
                t.completed                                ? 'Completed' :
                t.dueDate && t.dueDate < this.today        ? 'Overdue'   : 'Active';
            return {
                title:     t.title,
                listName:  list?.name  ?? null,
                listColor: list?.color ?? null,
                dueDate:   t.dueDate,
                status,
            };
        });
    });
}
```

## 10. Step 7 — Handle empty list selections

The backend requires `listSelections` to be `@NotEmpty`. When the user has no list filter applied (meaning "all lists"), the component must send **all** list IDs rather than an empty array.

In the `switchMap` in `DashboardComponent`:
```typescript
switchMap(([duration, status, listIds]) => {
    const allListIds = this.taskListStateService.lists().map(l => l.id);
    const effectiveListIds = listIds.length > 0 ? listIds : allListIds;
    // ...
})
```

> **Note:** This requires `TaskListStateService` to have already loaded before the first analytics fetch fires. Ensure `loadAll()` is called prior to the dashboard being shown (e.g., in a route resolver or the app shell).
